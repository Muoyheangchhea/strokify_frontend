Strokify Controller-Route Mismatch Fix

Fixing the function name mismatch between controller and routes

Issue Identified

Your controller exports predictStrokeController but your routes file is trying to import detectSymptoms.

This name mismatch is causing the 404 error because the route handler doesn't exist.

Solution

Option 1: Update your routes file (Recommended)

Change your routes/symptomRoutes.js to use the correct function name:

// routes/symptomRoutes.js - FIXED VERSION const express = require('express'); const router = express.Router(); const { predictStrokeController } = require('../controllers/symptomController'); // POST /api/symptoms/detect router.post('/detect', predictStrokeController); module.exports = router;

Option 2: Update your controller file

Alternatively, you could change your controller to export the expected function name:

// controllers/symptomController.js - ALTERNATIVE FIX // ... your existing code ... // Change the export to match what routes expects module.exports = { detectSymptoms: predictStrokeController };

Note: Option 1 is recommended as it's clearer and follows standard naming conventions.

Your Current Controller Code

Your controller exports predictStrokeController:

// controllers/symptomController.js // ... your code ... module.exports = { predictStrokeController }; // ‚Üê This exports predictStrokeController

Your Current Routes Code (Problem)

Your routes file tries to import detectSymptoms:

// routes/symptomRoutes.js (PROBLEM) const { detectSymptoms } = require('../controllers/symptomController'); // ‚Üë This tries to import detectSymptoms but it doesn't exist ‚Üë
